# Support

## Project Support

ZC-VIOS is an open-source project. For project questions, bug reports, documentation improvements, or feature requests, use GitHub issues where possible.

Please do not include passwords, payment details, private platform tokens, recovery codes, or sensitive personal information in GitHub issues.

---

## Bug Reports

Open an issue on GitHub with:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details
- Screenshots or logs if useful, with sensitive information removed

---

## Feature Requests

Open an issue describing:

- The problem you're trying to solve
- Your proposed solution
- How it fits with the existing system
- Any privacy, security, or maintenance concerns

---

## Documentation Improvements

Documentation improvements are welcome.

Keep content:

- factual,
- developer-focused,
- privacy-aware,
- and aligned with the current implemented state of the project.

---

## Security-Sensitive Reports

For security-sensitive reports, follow the guidance in `SECURITY.md`.

Do not post exploit details, credentials, private tokens, or sensitive data in public issues.

---

## Supporting Development

Optional development support is separate from project security, account access, or product functionality.

If you want to support continued development, use the public support link provided by the project owner in approved public profiles or project pages.

Support is optional and does not affect access to the open-source repository.
